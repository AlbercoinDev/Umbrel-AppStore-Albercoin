Scanning & Compute

Compute backend
GPU acceleration for silent payments scanning. Set to AUTO on machines with NVIDIA, Intel iGPU, or AMD GPU.
GPU batch size
Rows per GPU dispatch. Reduce (e.g. 10000-50000) if scanning hangs on older GPUs.
DuckDB threads
CPU cores used by DuckDB during CPU scanning. Empty = use all cores.

Memory

ScriptPubKey cache
~4 GB RAM per 10M entries. Reduce on low-RAM machines.
DuckDB memory limit
Max RAM for DuckDB (e.g. 2GB, 4096MB). Default: 80% of system RAM.

Bitcoin Core RPC

RPC request timeout
Seconds before RPC timeout. Raise for remote/slow Bitcoin Core nodes.
RPC batch size
Max sub-requests per JSON-RPC batch during initial mempool indexing.
Index start height
Block height to start indexing from. Default: Taproot activation (709632 mainnet).

How to Change Settings

To persist changes, edit the app environment variables and restart:

1. SSH into your Umbrel

2. Open ~/umbrel/app-data/albercoin-store-frigate/docker-compose.yml

3. Find the server service's environment section

4. Change values (e.g. FRIGATE_COMPUTE_BACKEND: "AUTO")

5. Run ~/umbrel/scripts/app restart albercoin-store-frigate

Or set these variables in Umbrel's app configuration if your version supports it.
